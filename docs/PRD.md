# 远程员工工作行为与代码风险监控系统 PRD

## 1. 文档信息

- 产品名称：远程员工工作行为与代码风险监控系统
- 文档版本：v0.1
- 适用阶段：MVP 需求定义与技术方案评审
- 目标用户：公司管理者、项目负责人、安全/审计负责人、远程全职员工
- 运行前提：员工使用公司提供的电脑，通过远程控制方式登录公司电脑工作；本系统不开发远控能力，只监控公司电脑上的工作会话。

## 2. 背景

公司存在远程全职研发员工，员工通过远控软件连接公司电脑完成日常开发。管理侧希望了解员工在工作时间内是否持续推进工作，并及时发现代码泄露、异常下载、长时间无操作等风险。

传统项目管理系统只能看到任务、提交、PR 等结果数据，无法解释员工在工作会话中的实际状态。单纯依赖远控实时查看又不可规模化，也缺少审计、统计和事件记录。

因此需要建设一个部署在公司电脑上的轻量客户端，定期采集屏幕和工作状态，在后台进行截图分析、前后截图变化对比、连续无变化识别、行为事件记录和代码泄露风险提示。

## 3. 产品定位

本系统不是远控软件，也不是隐蔽监控工具。

本系统定位为：

> 面向公司设备的远程研发工作会话记录、截图分析、静止风险识别和代码泄露风险预警系统。

核心目标：

1. 让管理者知道远程员工在公司电脑上是否处于有效工作会话。
2. 通过截图分析识别员工大致在做什么。
3. 通过连续截图对比发现长时间无变化、锁屏、断连、桌面停留等风险。
4. 通过 GitHub 和本机文件行为识别代码泄露风险。
5. 为异常情况提供可追溯的记录，而不是仅靠人工主观判断。

## 4. 产品原则

### 4.1 只监控公司电脑

系统只部署在公司提供的电脑上，只采集公司设备上的远程工作会话，不采集员工个人电脑、家庭网络、手机或私人设备。

### 4.2 透明告知

员工应提前知道公司电脑安装了监控客户端，知道采集内容、用途、保存期限、查看权限和申诉方式。

### 4.3 记录行为证据，不直接做人格判断

系统输出的是事实、趋势和风险事件，例如“连续 6 次截图无明显变化”“当前前台应用为浏览器”“GitHub 仓库被大量拉取”。系统不直接输出“员工摸鱼”“员工泄密”等最终结论。

### 4.4 不记录具体按键内容

系统可以记录键盘活动次数、鼠标活动次数、是否有输入，但不记录具体按键内容，不保存密码、Token、聊天全文和业务敏感输入。

### 4.5 原图短期保存，结构化结果长期保存

截图属于高敏感证据。MVP 默认保存 7-30 天，长期保存的是统计结果、事件记录、风险级别和分析摘要。

## 5. 目标与非目标

### 5.1 目标

MVP 需要支持：

1. 公司电脑客户端定时截图。
2. 前台应用、窗口标题、键鼠活动计数采集。
3. 截图和上一张截图做变化对比。
4. 连续 N 次无明显变化时在后台生成记录。
5. 对截图进行 OCR 和视觉分类，识别当前大致工作内容。
6. 后台按员工、设备、日期查看工作时间线。
7. 后台展示静止风险事件、锁屏/断连事件、异常应用事件。
8. 接入 GitHub 基础数据，展示提交、PR、仓库访问和异常拉取风险。
9. 具备权限控制和操作审计。

### 5.2 非目标

MVP 不做：

1. 远控功能本身。
2. 员工私人电脑监控。
3. 摄像头、麦克风采集。
4. 具体按键内容记录。
5. 私人聊天全文采集。
6. 自动绩效评分、自动扣款、自动处分。
7. 复杂 AI 员工画像。
8. 对所有截图无差别调用大模型分析。

## 6. 用户角色

### 6.1 系统管理员

职责：

- 管理员工、设备、项目、采集策略。
- 配置截图间隔、连续无变化阈值、保存期限。
- 管理角色权限。
- 查看系统运行状态和客户端在线状态。

### 6.2 项目负责人/主管

职责：

- 查看团队成员工作会话。
- 查看连续无变化、锁屏、断连、异常应用等风险事件。
- 查看截图分析摘要。
- 对异常事件进行复核和备注。

### 6.3 安全/审计负责人

职责：

- 查看代码泄露相关风险。
- 查看 GitHub 异常访问、仓库拉取、权限变更。
- 查看高风险截图和文件行为证据。
- 审计管理员和主管的数据访问行为。

### 6.4 员工

职责：

- 使用公司电脑远程办公。
- 可查看自己的工作会话记录和异常标记。
- 可对异常事件提交说明或申诉。

## 7. 业务场景

### 7.1 判断远程员工是否持续工作

主管进入后台，选择某员工和某一天，查看时间线：

- 09:00-09:30：IDE 编码，有明显变化。
- 09:30-09:45：浏览 GitHub PR，有明显变化。
- 09:45-09:52：截图连续无变化，前台应用为桌面，无键鼠活动。
- 09:52-10:20：腾讯会议，画面变化低，但会议软件活跃。

主管可以看到每段状态、截图摘要、变化级别和风险提示。

### 7.2 连续 N 次截图无变化

系统每 10 秒截图一次。若连续 6 次截图都判定为无明显变化，则生成一条静止风险事件：

- 员工：张三
- 设备：DEV-PC-001
- 时间：10:21:30 - 10:22:30
- 连续次数：6
- 前台应用：Chrome
- 窗口标题：GitHub - Pull Request
- 键鼠活动：无
- 判断原因：SSIM 0.992，变化块比例 0.7%，pHash 距离 2
- 风险级别：中

如果持续 5 分钟以上，则升级为高风险事件。

### 7.3 分析当前在做什么

系统对截图进行 OCR 和规则分析：

- 识别 Visual Studio Code、Cursor、JetBrains、Terminal、GitHub、Jira、飞书、企业微信、会议软件等。
- 提取窗口标题、浏览器标签、代码文件名、PR 标题、任务标题。
- 输出活动类型：编码、代码评审、文档、沟通、会议、网页浏览、空闲、锁屏、未知。

管理端展示的是活动摘要，不默认展示完整 OCR 文本。

### 7.4 代码泄露风险预警

安全负责人查看风险页：

- 员工短时间内 clone 多个仓库。
- 非工作时间大量 fetch。
- 插入 USB 设备后复制代码目录。
- 创建大压缩包，随后打开网盘或邮箱网页。
- 离职前一周访问大量非当前项目仓库。

系统生成风险事件，关联截图、文件行为、GitHub 操作和时间线。

## 8. 功能需求

### 8.1 员工与设备管理

#### 8.1.1 员工管理

字段：

- 员工姓名
- 员工编号
- 部门
- 岗位
- 直属主管
- 工作时间规则
- GitHub 账号
- 状态：在职、停用、离职

#### 8.1.2 设备管理

字段：

- 设备名称
- 主机名
- 操作系统
- 设备归属员工
- Agent 版本
- 最后心跳时间
- 屏幕数量
- 状态：在线、离线、异常、停用

### 8.2 客户端采集

#### 8.2.1 截图采集

需求：

- 支持定时截图。
- 默认间隔：10 秒。
- 可按部门、员工、设备配置间隔。
- 支持多屏幕截图。
- 支持截图压缩。
- 支持断网本地缓存和恢复补传。
- 支持截图失败重试。

截图元数据：

- 设备 ID
- 员工 ID
- 截图时间
- 屏幕编号
- 分辨率
- 前台应用
- 窗口标题
- 键盘活动次数
- 鼠标移动/点击次数
- 是否锁屏
- 是否远程会话连接中

#### 8.2.2 前台应用采集

需求：

- 采集前台进程名。
- 采集窗口标题。
- 采集应用路径或签名信息。
- 识别常见应用类型。

应用类型：

- IDE
- Terminal
- Browser
- Git Client
- Office/Document
- Chat
- Meeting
- Remote Desktop
- File Manager
- Compression Tool
- Cloud Drive
- Unknown

#### 8.2.3 键鼠活动采集

需求：

- 记录采集周期内键盘事件数量。
- 记录采集周期内鼠标移动、点击、滚轮事件数量。
- 不记录具体按键值。
- 不记录输入内容。
- 不记录剪贴板全文。

#### 8.2.4 本机文件风险采集

MVP 可选，建议作为第二阶段或安全模块开启：

- 代码目录大量复制。
- 大文件压缩包创建。
- USB 插入。
- 文件复制到 USB。
- 打开网盘客户端或网页。
- 邮箱附件上传行为。

### 8.3 截图变化检测

每张截图需与同设备、同屏幕的上一张截图对比。

检测指标：

- pHash/dHash 距离。
- SSIM 结构相似度。
- 分块变化比例。
- 有效变化区域数量。
- 光标/时钟/小通知过滤结果。

变化级别：

- `none`：无明显变化。
- `minor`：微小变化。
- `major`：明显变化。
- `unknown`：无法判断。

有效变化规则：

- 窗口切换、页面滚动、代码变化、文档变化、终端输出变化属于有效变化。
- 鼠标光标闪烁、输入光标闪烁、系统时间变化、小角标变化不应被认定为有效变化。

### 8.4 连续无变化事件

系统需要配置连续无变化阈值 N。

默认规则：

- 截图间隔：10 秒。
- 连续无变化阈值：N=6。
- 触发时间：约 60 秒。
- 高风险阈值：连续 5 分钟以上。

事件生命周期：

1. 第一次无变化，记录普通 `screen_unchanged`。
2. 连续无变化次数达到 2，标记为静止区间开始。
3. 连续无变化次数达到 N，生成 `no_change_streak_triggered`。
4. 如果持续无变化，更新事件持续时间和 streak_count。
5. 一旦出现明显变化，关闭事件，记录结束时间。

风险级别建议：

- 低：连续 1-2 分钟无变化，但存在键鼠活动或会议/文档场景。
- 中：连续 1-5 分钟无变化，且无键鼠活动。
- 高：连续超过 5 分钟无变化，且处于桌面、锁屏、娱乐网站、无关应用或无键鼠活动。

### 8.5 截图内容分析

#### 8.5.1 活动类型识别

系统应输出：

- activity_type
- active_app
- page_topic
- confidence
- evidence_summary

活动类型：

- coding：编码
- code_review：代码评审
- terminal：终端/命令行
- debugging：调试
- documentation：文档编写/阅读
- meeting：会议
- communication：沟通
- project_management：任务管理
- browsing_work_related：工作相关网页
- browsing_unrelated：疑似无关网页
- idle_desktop：桌面空闲
- locked：锁屏
- disconnected：远控断连
- unknown：未知

#### 8.5.2 OCR 分析

OCR 需要提取：

- 窗口标题。
- 浏览器标签。
- URL 域名，不默认保存完整 URL 查询参数。
- IDE 项目名。
- 代码文件名。
- PR/Issue/任务标题。
- 主要页面标题。

OCR 文本默认做摘要，不长期保存全文。

#### 8.5.3 低置信度复核

当规则和 OCR 无法判断时，标记为低置信度：

- confidence < 0.6。
- activity_type = unknown。
- 同一员工同一时段连续未知。

低置信度截图可进入人工复核或多模态模型复判队列。

### 8.6 GitHub 风险监控

MVP 接入 GitHub Organization/Enterprise 数据。

基础指标：

- commit 数。
- PR 创建、更新、合并。
- review 参与。
- issue/comment 活动。
- 仓库访问记录。
- clone/fetch 事件。
- 成员和权限变更。
- deploy key、SSH key、token 相关事件。

风险规则：

- 短时间内 clone 多个仓库。
- 访问非负责项目仓库。
- 非工作时间大量 fetch。
- 合作/任职结束后仍尝试访问。
- 权限突然提升。
- secret scanning 告警。
- 仓库可见性变更。

## 9. 数据权限

### 9.1 权限分层

- 员工：查看自己的摘要和异常记录。
- 主管：查看直属团队数据。
- 安全负责人：查看安全风险事件。
- 系统管理员：管理配置和设备，不默认查看所有原图。
- 审计员：查看操作日志和访问记录。

### 9.2 操作审计

必须审计：

- 登录后台。
- 查看员工详情。
- 查看原图。
- 导出截图。
- 修改策略。
- 删除数据。
- 标记复核结果。
- 修改员工/设备权限。

## 10. 数据留存

默认建议：

- 截图原图：7-30 天。
- 缩略图：30-90 天。
- OCR 摘要：90-180 天。
- 行为事件：1-2 年。
- 审计日志：1-3 年。

具体期限应按公司制度和当地法律要求确认。

## 11. 风险与误判处理

### 11.1 常见误判

- 看文档长时间不动。
- 会议时画面变化少。
- 等待编译或测试。
- 观察日志。
- 远控卡顿或断连。
- 浏览技术文档但变化少。

### 11.2 降低误判的方法

- 结合前台应用。
- 结合键鼠活动计数。
- 结合 GitHub 和任务活动。
- 对会议、文档、日志场景降低静止风险级别。
- 低置信度事件进入人工复核。
- 允许员工提交说明。

## 12. 验收标准

### 12.1 客户端采集

- 能按配置间隔稳定截图。
- 能上传截图和元数据。
- 能记录前台应用、窗口标题、键鼠活动计数。
- 断网后能本地缓存并恢复补传。

### 12.2 变化检测

- 能正确识别相同画面。
- 能过滤光标闪烁和小范围系统时间变化。
- 能识别窗口切换、页面滚动、代码变化。
- 连续 N 次无变化能生成事件。
- 出现明显变化后能结束静止事件。

### 12.3 活动识别

- 能识别常见 IDE、浏览器、GitHub、会议、聊天、文档、终端。
- 每条分析结果有置信度。
- 低置信度记录不会直接作为高风险结论。

### 12.4 后台

- 能查看员工时间线。
- 能查看事件中心。
- 能查看前后截图对比。
- 能修改策略。
- 能审计原图查看和导出。

### 12.5 GitHub 风险

- 能拉取基础 GitHub 活动。
- 能关联员工 GitHub 账号。
- 能识别异常 clone/fetch 或权限变更。
- 能在事件中心展示 GitHub 风险事件。
